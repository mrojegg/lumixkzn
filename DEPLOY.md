# Deploy Checklist

## 1. Репозиторий

- Создать репозиторий на GitHub или GitLab
- Залить содержимое этой папки в ветку `main`

## 2. Публикация

Текущая рекомендуемая схема для этого проекта:

- GitHub Pages
- кастомный домен с REG.RU
- SSL от GitHub Pages после привязки домена

## 3. Файлы сайта

Основной файл:

`index.html`

Для GitHub Pages уже подготовлены:

- `.github/workflows/deploy-pages.yml`
- `robots.txt`
- `sitemap.xml`

## 4. Домен

После покупки домена на REG.RU:

- добавить файл `CNAME` с доменом
- прописать DNS домена на GitHub Pages
- проверить открытие сайта по домену

## 5. GitHub Pages

- включить Pages в репозитории
- источник: `GitHub Actions`
- проверить успешный workflow после `push`

## 6. Финальная проверка

- сайт открывается по `https`
- Pages workflow завершился без ошибок
- адаптив проверен
- ошибок в консоли нет
- реквизиты и контакты актуальны
